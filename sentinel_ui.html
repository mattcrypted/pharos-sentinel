      How it works
      
Follow one check through the gate.

      
Every verdict is built from read-only Foundry cast reads of the live chain — no indexer, no third-party API.

    


    

      
01

      

        
Read the chain

        
Sentinel pulls the target's bytecode, proxy storage slots, and token calls — all read-only, all through Foundry.

        

          
cast code 0x75fb…35d515 bytes

          
cast storage …impl slot (EIP-1967)read

          
cast call "paused()(bool)"true

        

      

    


    

      
02

      

        
Detect the signals

        
It walks the opcodes and proxy slots for the patterns that actually drain wallets — not surface heuristics.

        

          
SELFDESTRUCT in bytecodefound · +25

          
DELEGATECALL outside a proxyfound · +15

          
owner() is an EOAfound · +10

          
contract pausedfound · +20

        

      

    


    

      
03

      

        
Score it

        
Signals stack into a single additive score, so multiple small risks add up to a loud one.

        
25 + 15 + 10 + 20= 70

      

    


    

      
04

      

        
The verdict

        
A band turns the score into a decision an agent can act on directly.

        

          
SAFE
score < 35

          
CAUTION
35 – 69

          
DANGEROUS
≥ 70

        

      

    


    

      
05

      

        
Always the reasons

        
Never just a number. Every verdict carries the human-readable why — the audit trail behind the call.

        

          
▸ bytecode contains SELFDESTRUCT — can be destroyed

          
▸ unguarded DELEGATECALL — can run external code

          
▸ single EOA owner holds privileged control

        

      

    


    

      
06

      

        
Gate the action

        
execution_plan turns the verdict into approve/block plus a de-risked size — so value moves only when it's safe to.

        

          
safe target · transferAPPROVED → full size

          
dangerous target · approveBLOCKED → no tx signed
